# QQ 农场科研版：Mac 统一调度流程

## 最终目标

你只在 Mac 上工作。

所有其他电脑像“科研农场”里的地块一样自动运行：每台电脑都有一个 EvoScientist 循环工程 Agent，自己领取科研任务、执行实验、分析结果、尝试修复低价值问题。只有遇到 AI 解决不了、必须由你判断的高级问题时，才回到 Mac 上请求你批准或决策。

最终形态：

```text
你负责：
  授权、方向、边界条件、科研判断、数据飞轮升级

Agent 负责：
  执行、监控、重试、总结、低价值问题处理、结果沉淀

Mac 负责：
  统一对话、统一审批、统一查看状态、统一调度

其他电脑负责：
  自动跑任务，主动汇报，不主动打扰
```

## 一条必须打通的主流程

```text
Mac 对话入口
  -> 控制中心
  -> 分配任务给某台电脑
  -> EvoScientist Agent 执行科研任务
  -> Agent 自己处理低价值问题
  -> 重要问题回到 Mac 请求你
  -> 你批准/修改方向/设定边界
  -> Agent 继续执行
  -> 结果进入科研数据飞轮
```

这个流程打通之后，其他所有功能都是增强项。

## 你在 Mac 上看到什么

Mac 上不应该是一堆远程桌面窗口，而应该是一个统一操作台。

第一版只需要这些：

```text
1. 对话输入框
2. 当前所有机器状态
3. 正在运行的任务
4. 失败/阻塞的任务
5. 等你批准的问题
6. 最近产出的结果摘要
7. 可一键远控的入口
```

你不需要关心任务在哪台电脑跑，除非你想指定。

示例输入：

```text
把昨天失败的参数扫描任务重新跑一遍，优先使用空闲 GPU。
失败一次先自动修复，失败两次再来找我。
如果涉及删除数据、重启机器、改许可证配置，必须先问我。
```

## Agent 应该自动处理的低价值问题

这些问题不应该打扰你：

```text
任务进程挂了，自动重启一次
Python 包缺失，安装白名单里的包
日志太长，自动摘要
结果文件太多，自动打包
机器忙，自动换一台
网络短暂断开，恢复后继续上报
一次随机失败，自动重试
磁盘空间不足但能清理临时目录，自动清理
仿真参数格式小错误，按模板修正
```

目标：低价值问题由 Agent 自己消化。

## 必须找你的高级问题

这些问题必须回到 Mac 上找你：

```text
是否授权某个软件/账号/API
是否扩大任务权限
是否删除重要数据
是否重启电脑或停止大量任务
是否改变实验方向
是否修改边界条件
是否接受一个不确定的实验假设
是否继续烧大量算力
是否切换模型、方法或研究路线
是否远控进入某台电脑人工检查
```

你的价值不在于点按钮和看日志，而在于做这些高级判断。

## 人类请求的标准格式

Agent 找你时，不能只说“出错了”。它必须给出一个可决策请求。

每个请求必须包含：

```text
1. 发生在哪台机器
2. 正在执行什么任务
3. 问题摘要
4. Agent 已经尝试了什么
5. 为什么需要你
6. 推荐方案
7. 备选方案
8. 每个方案的风险
9. 批准后有效范围
```

示例：

```text
任务：高温边界条件参数扫描
机器：lab-gpu-02
问题：当前边界条件导致数值发散，Agent 已尝试降低步长和重启任务，仍失败。

需要你决定：
  A. 缩小温度范围继续扫描
  B. 保持范围但换求解器
  C. 停止任务，标记该参数区间为不可行

Agent 推荐：
  A。风险最低，能保留实验连续性。

是否批准？
```

## 科研数据飞轮

QQ 农场科研版的长期价值不是远控，而是数据飞轮。

```text
任务执行
  -> 结果收集
  -> 失败归因
  -> 人类决策记录
  -> 规则/模板更新
  -> Agent 下次少问一次
  -> 更多任务自动化
  -> 更高质量科研数据
  -> 更强 Agent
```

每次你解决高级难题，都应该沉淀成系统能力：

| 你解决的问题 | 沉淀成什么 |
| --- | --- |
| 授权一次软件 | 授权流程模板 |
| 判断一次边界条件 | 边界条件规则 |
| 修复一次依赖问题 | 自动修复脚本 |
| 选择一次实验方向 | 研究策略记录 |
| 解释一次失败原因 | 失败分类器 |
| 批准一次高风险操作 | 带作用域的权限策略 |

目标不是每次都人工救火，而是让系统每次学会一点。

## 第一版必须实现的闭环

第一版不要贪多，只打通这个闭环：

```text
1. 你在 Mac 上输入任务
2. 控制中心记录任务
3. 某台电脑的 EvoScientist Agent 收到任务
4. Agent 执行一个真实科研脚本
5. Agent 上报日志摘要
6. Agent 遇到一个需要批准的问题
7. Mac 显示批准请求
8. 你批准或拒绝
9. Agent 根据你的决定继续
10. 最终结果进入结果库
```

完成这个闭环，QQ 农场科研版就已经成立。

## 最小产品定义

第一版产品只需要这些模块：

```text
Mac 页面：
  对话、机器列表、任务列表、审批队列、结果摘要

控制中心：
  任务分发、状态记录、审批记录、审计日志

EvoScientist Agent：
  收任务、跑任务、报状态、请求批准、上传结果

远控兜底：
  RustDesk
```

不需要一开始就做：

```text
复杂 Mac App
自研远程桌面
Kubernetes
完整监控平台
复杂多用户权限
过度漂亮的界面
```

## 项目原则

```text
1. Mac 是人类入口，不是服务器。
2. Agent 主动汇报，不等人远控。
3. 低价值问题自动处理。
4. 高价值问题才找你。
5. 每次人工决策都要沉淀为规则、模板或数据。
6. 远控只做兜底，不做主流程。
7. 系统越用越少打扰你。
```

## 最终一句话

QQ 农场科研版不是“远控很多电脑”。

它是：

```text
一个由 Mac 统一调度的科研自动化农场。
其他电脑自动种地、施肥、除草、收获。
Agent 处理日常琐事。
你只处理授权、方向、边界条件和科研判断。
每次判断都会进入数据飞轮，让下一轮循环工程更高效。
```
